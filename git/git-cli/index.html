GitHub CLI Study

GitHub 복사하기

GitHub의 프로젝트를 복사합니다.


GitHub 연동하기

로컬의 Git과 서버의 GitHub와 연동합니다.
(주의) GitHub의 프로젝트 이름과 Git의 폴더명이 같아야 합니다.


GitHub master 브랜치 이름 변경

브랜치의 이름을 master에서 main으로 변경합니다.
(이유) 주인(master)과 노예(slave)의 언어 순화를 위해 변경합니다.


GitHub에 반영하기

Git의 기록을 GitHub에 반영합니다.


GitHub에 새로운 내용 확인하기

GitHub에 새로운 내용이 저장되어 있는지 확인합니다.

(주의) Git에 변경사항이 없다면 문제가 되지 않지만,
변경사항이 있는 경우 GitHub에 저장하려고하면
Git과 GitHub의 내용이 같지 않아서 동기화를 해야 합니다.
